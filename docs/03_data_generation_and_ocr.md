# 虚拟数据、OCR 与脱敏方案

## 1. 设计原则

由于真实快递面单涉及隐私，且短时间内难以收集足够数据，本项目采用虚拟快递面单作为主要数据来源。

虚拟数据需要满足：

- 字段真实可信，但不能使用真实个人隐私。
- 图片格式接近快递面单。
- 字段位置相对稳定，方便 OCR 和打码演示。
- 同步生成标签 JSON，方便测试字段提取效果。

## 2. 数据目录

```text
dataset/
├── templates/
│   ├── template_01.png
│   └── template_02.png
├── generated/
│   ├── waybill_0001.jpg
│   ├── waybill_0002.jpg
│   └── ...
└── labels/
    ├── waybill_0001.json
    ├── waybill_0002.json
    └── ...
```

## 3. 虚拟面单字段

每张快递面单至少包含：

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| receiver_name | 张三 | 收件人姓名 |
| phone | 13812345678 | 收件人手机号 |
| address | 福建省泉州市丰泽区测试路88号 | 收件地址 |
| tracking_number | SF1234567890123 | 快递单号 |
| sender_name | 李四 | 寄件人姓名，可选 |
| sender_phone | 13911112222 | 寄件人手机号，可选 |
| company | 顺丰速运 | 快递公司名称，可选 |

MVP 阶段优先识别收件人信息，寄件人信息可以作为模板装饰或增强项。

## 4. 标签 JSON 格式

每张生成图片对应一个 JSON 标签文件。

```json
{
  "image": "waybill_0001.jpg",
  "receiver_name": "张三",
  "phone": "13812345678",
  "address": "福建省泉州市丰泽区测试路88号",
  "tracking_number": "SF1234567890123",
  "fields": [
    {
      "name": "receiver_name",
      "text": "收件人：张三",
      "box": [[50, 80], [220, 80], [220, 115], [50, 115]]
    },
    {
      "name": "phone",
      "text": "电话：13812345678",
      "box": [[50, 125], [310, 125], [310, 160], [50, 160]]
    },
    {
      "name": "address",
      "text": "地址：福建省泉州市丰泽区测试路88号",
      "box": [[50, 170], [620, 170], [620, 215], [50, 215]]
    },
    {
      "name": "tracking_number",
      "text": "运单号：SF1234567890123",
      "box": [[50, 35], [390, 35], [390, 70], [50, 70]]
    }
  ]
}
```

这个标签既可以用于验证字段提取，也可以在 mock OCR 阶段直接转换成 OCR 结果。

## 5. 生成脚本要求

脚本路径建议：

```text
scripts/generate_waybills.py
```

脚本职责：

- 读取 `dataset/templates/` 下的模板图片。
- 随机生成虚拟姓名、手机号、地址、快递单号。
- 使用 Pillow 将文字绘制到模板固定位置。
- 保存生成图片到 `dataset/generated/`。
- 保存标签 JSON 到 `dataset/labels/`。

建议参数：

```text
--count 50
--template template_01.png
--output-dir dataset/generated
--label-dir dataset/labels
```

示例命令：

```bash
python scripts/generate_waybills.py --count 50
```

## 6. 模板文本布局建议

为了便于 OCR 和字段抽取，模板中建议固定关键词：

```text
运单号：SF1234567890123
收件人：张三
电话：13812345678
地址：福建省泉州市丰泽区测试路88号
```

关键词尽量不要频繁变化。比如手机号字段统一用“电话：”，不要一会儿写“手机：”，一会儿写“联系电话：”。这样能降低正则提取难度。

## 7. OCR 结果统一格式

无论是 mock OCR 还是 PaddleOCR，最终都必须转换成统一格式：

```json
[
  {
    "text": "收件人：张三",
    "confidence": 0.98,
    "box": [[50, 80], [220, 80], [220, 115], [50, 115]]
  },
  {
    "text": "电话：13812345678",
    "confidence": 0.97,
    "box": [[50, 125], [310, 125], [310, 160], [50, 160]]
  }
]
```

系统内部只依赖这个格式，不直接依赖 PaddleOCR 原始返回结构。

## 8. Mock OCR 方案

第一阶段不接 PaddleOCR，先通过 mock OCR 跑通闭环。

mock OCR 可以采用两种方式：

### 8.1 固定返回

不读取图片内容，直接返回固定 OCR 文本：

```json
[
  {
    "text": "运单号：SF1234567890123",
    "confidence": 0.99,
    "box": [[50, 35], [390, 35], [390, 70], [50, 70]]
  },
  {
    "text": "收件人：张三",
    "confidence": 0.98,
    "box": [[50, 80], [220, 80], [220, 115], [50, 115]]
  },
  {
    "text": "电话：13812345678",
    "confidence": 0.97,
    "box": [[50, 125], [310, 125], [310, 160], [50, 160]]
  },
  {
    "text": "地址：福建省泉州市丰泽区测试路88号",
    "confidence": 0.96,
    "box": [[50, 170], [620, 170], [620, 215], [50, 215]]
  }
]
```

### 8.2 读取 label

如果上传图片来自 `dataset/generated/`，可以根据图片名找到对应 label JSON，并将 label 中的 `fields` 转换成 OCR 结果。

这种方式更适合测试多张虚拟面单。

## 9. PaddleOCR 接入方案

第二阶段再接入 PaddleOCR。

`ocr_service.py` 内部可以保留统一入口：

```python
def run_ocr(image_path):
    if settings.OCR_MODE == "mock":
        return run_mock_ocr(image_path)
    return run_paddle_ocr(image_path)
```

PaddleOCR 原始结果需要转换为：

```python
{
    "text": text,
    "confidence": float(score),
    "box": box
}
```

这样后续字段提取和图片脱敏模块不需要关心 OCR 来源。

## 10. 字段提取规则

字段提取输入为 OCR 结果列表，输出为结构化字段。

### 10.1 文本合并

先将 OCR 文本合并：

```text
运单号：SF1234567890123
收件人：张三
电话：13812345678
地址：福建省泉州市丰泽区测试路88号
```

### 10.2 手机号

正则：

```text
1[3-9]\d{9}
```

### 10.3 快递单号

可先支持常见规则：

```text
SF\d{10,15}
YT\d{10,15}
ZTO\d{10,15}
YD\d{10,15}
\d{10,18}
```

### 10.4 收件人

优先匹配：

```text
收件人[:：]\s*([^\s，,；;]+)
```

如果姓名和手机号在同一行，要避免把手机号也吃进去。

### 10.5 地址

优先匹配：

```text
地址[:：]\s*(.+)
```

地址通常可能很长，允许包含省、市、区、街道、门牌号等内容。

## 11. 字符串脱敏

手机号脱敏规则：

```text
13812345678 -> 138****5678
```

实现逻辑：

```python
def mask_phone(phone):
    if not phone or len(phone) != 11:
        return phone
    return phone[:3] + "****" + phone[-4:]
```

地址脱敏可以作为增强项，例如：

```text
福建省泉州市丰泽区测试路88号 -> 福建省泉州市丰泽区***
```

MVP 阶段只要求手机号结构化脱敏，地址可以完整展示，也可以只在图片中打码。

## 12. 图片脱敏策略

### 12.1 MVP 策略

第一版使用黑色矩形遮挡敏感文本区域。

敏感字段包括：

- 收件人姓名
- 手机号
- 地址

快递单号是否脱敏可根据答辩口径决定。建议也可以部分遮挡，因为它属于物流隐私信息。

### 12.2 基于 OCR box 的策略

遍历 OCR 结果：

```text
如果 text 包含手机号
  -> 遮挡该 box
如果 text 包含“收件人”
  -> 遮挡该 box
如果 text 包含“地址”
  -> 遮挡该 box
```

对于每个 box，取最小外接矩形：

```text
x_min = min(x)
y_min = min(y)
x_max = max(x)
y_max = max(y)
```

然后使用 Pillow 绘制矩形：

```python
draw.rectangle([x_min, y_min, x_max, y_max], fill="black")
```

### 12.3 马赛克增强项

如果时间允许，可以把黑色矩形改成马赛克：

1. 裁剪敏感区域。
2. 将区域缩小。
3. 再放大回原尺寸。
4. 粘贴回原图。

这种效果更像真实隐私保护系统，但不是 MVP 必需项。

## 13. 测试样例

至少准备 5 张虚拟面单：

| 编号 | 场景 | 预期 |
| --- | --- | --- |
| 1 | 标准字段齐全 | 四个核心字段全部识别 |
| 2 | 地址较长 | 地址能完整提取或部分提取 |
| 3 | 不同快递单号前缀 | 单号能识别 |
| 4 | 图片轻微倾斜 | OCR 可能波动，mock 阶段不要求 |
| 5 | 缺少手机号 | 接口成功，phone 为 null |

## 14. 答辩讲法

可以这样描述：

```text
由于快递面单包含个人隐私，本项目使用脚本生成虚拟面单数据，避免真实隐私泄露。
系统先通过 OCR 获取文本和文本框，再使用规则方法提取关键字段。
对于手机号等敏感信息，系统在结构化结果中进行脱敏展示，并根据 OCR 定位结果对图片区域进行遮挡，从而实现识别与隐私保护的完整流程。
```
