## 3. 脱敏功能后续要求

当前 base 版本先用黑色矩形遮挡敏感区域。后续建议改成马赛克脱敏，并且做“部分保留”。

建议脱敏规则：

- 姓名：保留首尾，中间打码或马赛克。
  - 例：`张伟` 可以显示为 `张*`
  - 例：`欧阳志强` 可以显示为 `欧**强`
- 手机号：保留前三位和后四位。
  - 例：`13812345678` -> `138****5678`
- 地址：保留省市区大致区域，详细门牌、小区、楼栋、房间号打码。
  - 例：`广东省深圳市南山区科技园路168号创新公寓6栋2100室`
  - 可显示为：`广东省深圳市南山区***`
- 快递单号：可以保留部分数字，其余打码。
  - 例：`81793571802948` -> `8179******2948`

图片脱敏建议：

- 不再用纯黑块作为最终效果。
- 使用马赛克覆盖姓名、手机号、详细地址、快递单号等敏感区域。
- 如果能做到更细，可以只马赛克中间敏感字符，保留姓名首尾和手机号首尾。
- 如果时间不够，可以先对整块 OCR box 做马赛克，答辩时说明后续可以细化到字符级。

## 4. 为什么要做脱敏

快递面单包含大量个人隐私信息，例如：

- 收件人姓名
- 手机号
- 详细住址
- 快递单号
- 寄件人信息

这些信息如果未经处理直接展示或传播，可能造成：

- 电话骚扰
- 地址泄露
- 身份关联风险
- 包裹物流信息被他人查询
- 个人隐私被二次传播

所以本项目不仅要识别快递面单内容，还要在识别后进行隐私保护。这样系统的完整价值是：

```text
OCR 识别信息
  -> 提取关键字段
  -> 判断敏感内容
  -> 对结构化结果和图片同时脱敏
  -> 在保护隐私的前提下展示可用信息
```

答辩时可以这样讲：

```text
本系统并不是简单地把快递面单 OCR 出来，而是在识别后进一步判断哪些字段属于个人隐私，并对手机号、姓名、地址等信息进行脱敏处理，从而兼顾信息提取和隐私保护。
```

## 7. 旋转校正功能建议接口

如果实现图像预处理，建议后端接口在 `POST /api/recognize` 中增加一个表单参数：

```text
file: 图片文件
enable_preprocess: true/false
```

返回结果中增加：

```json
{
  "preprocessed_image_url": "/outputs/preprocessed_xxx.png",
  "preprocess": {
    "enabled": true,
    "applied": true,
    "angle": -2.6,
    "message": "已进行轻微倾斜校正"
  }
}
```

建议处理流程：

```text
用户上传图片
  -> 后端保存原图
  -> 如果 enable_preprocess=true，则进行轻微旋转校正
  -> 使用校正后的图片进行 OCR
  -> 从 OCR 文本中提取字段
  -> 根据 OCR box 生成马赛克脱敏图
  -> 前端展示识别结果、预处理信息和脱敏图
```

注意：

- clean/augmented 数据用于主流程和脱敏展示。
- rotated 数据用于图像校正和 OCR 鲁棒性测试。
- rotated 原始标签中的 box 不再准确，因此不要直接用它评估脱敏框位置。
- 如果 PaddleOCR 返回校正后图片上的新 box，就可以用新 box 做马赛克脱敏。
